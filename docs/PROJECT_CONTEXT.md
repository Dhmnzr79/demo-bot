# Project Context

Проект — Python/RAG-бот для сайта клиники. Сейчас это demo-prod бот для одного клиента, но в будущем проект может развиться в мультиклиентный продукт с тарифами, дашбордом, аналитикой, разными базами знаний и сценариями лидогенерации.

Автор проекта — маркетолог, копирайтер и UX-специалист, не программист. Поэтому технические решения должны объясняться простым языком: что меняем, зачем, какие риски, как это повлияет на масштабирование.

## Главные цели

- стабильный бот для клиента;
- качественные ответы на основе базы знаний;
- мягкая лидогенерация;
- управляемые CTA и follow-up сценарии;
- подготовка к будущей мультиклиентности;
- возможность в будущем добавить дашборд, тарифы, usage tracking и аналитику.

## Что важно

- не ломать текущего клиента;
- не хардкодить клиентскую логику в общем коде;
- не смешивать backend, frontend, content и business logic;
- не делать временные решения без явной пометки;
- учитывать конверсию, доверие и UX, а не только “чтобы код работал”.
